The `Address` type provides an address information on multiple EVM chains.

```graphql
type Address {
  address: AddressEVM!
  ethWallet: Wallet
  goerliWallet: Wallet
  identity: Identity!
  followers(
  offset: Int
  limit: Int
): FollowConnection!
  followings(
  offset: Int
  limit: Int
): FollowConnection!
  followStats: FollowStats!
  isFollowedBy(
  addresses: [AddressEVM!]!
): [Boolean!]!
  isFollowing(
  addresses: [AddressEVM!]!
): [Boolean!]!
}
```


### Fields

#### `address` ([`AddressEVM`](/reference/scalars/address-evm))

`address` the address.


#### `ethWallet` ([`Wallet`](/reference/objects/wallet))

`ethWallet` the wallet address on Ethereum.


#### `goerliWallet` ([`Wallet`](/reference/objects/wallet))

`goerliWallet` the wallet address on Goerli.


#### `identity` ([`Identity`](/reference/objects/identity))

`identity` provides information about a user's identity.


#### `followers` ([`FollowConnection`](/reference/objects/follow-connection))

`followers` provides a paginated list of ordered followers profiles.
- ##### `offset` ([`Int`](/reference/scalars/int))




- ##### `limit` ([`Int`](/reference/scalars/int))




#### `followings` ([`FollowConnection`](/reference/objects/follow-connection))

`followings` provides a paginated list of ordered followings profiles.
- ##### `offset` ([`Int`](/reference/scalars/int))




- ##### `limit` ([`Int`](/reference/scalars/int))




#### `followStats` ([`FollowStats`](/reference/objects/follow-stats))

`followStats` returns stats about a user's followers and followings


#### `isFollowedBy` ([`Boolean`](/reference/scalars/boolean))

`isFollowedBy` returns the `followed by` status of one or multiple addresses.
- ##### `addresses` ([`AddressEVM`](/reference/scalars/address-evm))




#### `isFollowing` ([`Boolean`](/reference/scalars/boolean))

`isFollowing` returns the `following` status to one or multiple addresses.
- ##### `addresses` ([`AddressEVM`](/reference/scalars/address-evm))
